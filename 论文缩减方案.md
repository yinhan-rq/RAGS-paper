## 方案二：§4 Implementation 大幅压缩（预计节省 ~0.4 页）⭐⭐⭐

### 2.1 KV Cache 前缀树描述精简（节省约 6 行）

**设计区分：**

本系统与 sglang 前缀树的**核心差异**在于引入了**全局 KV Cache 前缀树**（Global Prefix Cache Tree）：每个节点额外存储 `replica` 字段，将树节点映射至具体 LLM 实例，使全局调度器能跨实例定位可复用的 KV cache（`lines 1060–1066`）。这是本文独有的设计，**必须完整保留**。

Local Prefix Cache Tree 的三字段节点结构（`token sequence` / `ref` / `block`）与标准前缀树实现高度相似，属于工程细节，可以简化表述。

**操作：**
- 将 `\begin{itemize}...\end{itemize}` 的三个 bullet points 压缩为一句话内联描述，例如：*"Each node stores the token sequence, a reference count for memory reclamation, and a block index into the physical KV store."*
- **保留** Global Prefix Cache Tree 的完整描述（`replica` 字段及跨实例复用语义），这是与 sglang 等已有系统的关键区别
- 可在 Local/Global 树的对比引入处，以一句话点明二者与 sglang 的区别：*"Unlike sglang's single-level prefix tree, our design maintains both a per-instance local tree and a cluster-wide global tree augmented with replica mappings to enable cross-instance KV cache reuse."*

**参考行号：** `lines 1022–1066`

## 核心原则

1. **图已呈现的数值，正文无需重复推导** — §2 中所有数值算例均有对应图示
2. **实现层工程细节可大幅压缩** — §4 中树结构字段、内存对齐等细节不影响贡献评估
3. **与 Abstract / Introduction 重叠的内容删除** — §2.1 的特征描述与引言高度重叠
4. **标准 ML 流程公式可合并描述** — Transformer 前向传播公式链无需逐步展开
5. **同一节内的 Overview / 总结段属于冗余** — 读者不需要在同一节内读到两遍相同内容
